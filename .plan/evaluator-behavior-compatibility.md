# Evaluator behavior compatibility plan

## Non-negotiable constraints

- Preserve all JVM obfuscation invariants from `AGENTS.md`: no fallback behavior, no reduced transform coverage, no reflection/lambda special-case deobfuscation, no plaintext reflective application data, no static key exposure, and no skip-on-error path.
- Use fresh artifacts only. Do not rely on stale `build/test-jvm-full-obf-perf/` evaluator outputs as proof.
- Do not use Recaf. Use `javap`, ASM helper programs, JVM exception logging, and repository tests/CLI outputs.
- Do not touch unrelated current native-code/test work shown by `git status --short`.
- Do not create files or folders under `/tmp`.

## Current evidence

- The full JVM obfuscation gate currently obfuscates `test-jars/evaluator-unobf.jar` but does not execute it in `JvmFullObfuscationPerfTest`.
- User reports evaluator behavior inconsistency after the reflection/lambda compatibility repair.
- The repository Gradle wrapper has already been approved for this repair stream.

## Subtasks

[ ] EVAL-1: Regenerate fresh evaluator artifacts and compare behavior

- Scope: Freshly obfuscate evaluator with `test-jars/full-jvm-obf.yml`, identify the evaluator entry point and required runtime invocation, then run original and full-obfuscated evaluator artifacts under the same inputs. Do not modify transform code in this subtask.
- Required evidence: Fresh obfuscation command output, original/full-obfuscated stdout/stderr/exit code, exact input arguments or stdin used, and exact first behavioral divergence.
- Validation command/runtime target: Use repository `./gradlew` or installed CLI from a fresh test run to generate `build/test-jvm-full-obf-perf/evaluator-full-jvm-obf.jar`, then run original and obfuscated evaluator with identical JVM args and inputs.
- Completion criteria: A concrete evaluator divergence is reproduced from fresh artifacts and recorded without relying on stale output.

[ ] EVAL-2: Localize evaluator root cause

- Scope: Map the divergent evaluator behavior to transformed class/method/descriptor/callsite/key path using `javap`, ASM inspection helpers, and JVM exception/logging output. Do not implement until the failing invariant is proven.
- Required evidence: Exact transformed method/site, generated bytecode or metadata proving the incorrect invariant, and source transform owner responsible for the defect.
- Validation command/runtime target: Focused original/full-obf evaluator reruns plus targeted disassembly/ASM dumps of the affected classes.
- Completion criteria: Root cause is generic and architecture-level, not evaluator-specific.

[ ] EVAL-3: Patch generic JVM transform compatibility

- Scope: Fix the proven transform invariant at the source while preserving CFF, method-parameter carriers, hidden keys, invokedynamic, string/constant obfuscation, and renaming strength. No fallback, skip, helper-layer bypass, or plaintext exposure.
- Required evidence: Source-level mapping from the failing transformed site to the generic transform change.
- Validation command/runtime target: Run the smallest focused command that exercises the changed evaluator path, then run the full JVM obfuscation gate.
- Completion criteria: Fresh obfuscated evaluator matches original behavior under the recorded inputs and no forbidden weakening is introduced.

[ ] EVAL-4: Final evaluator parity and checkpoint

- Scope: Regenerate the full JVM obfuscation artifacts, run evaluator original/full-obf parity, run related full-obf fixtures, update this plan with evidence, and commit only evaluator-fix source plus this plan update.
- Required evidence: Fresh evaluator parity logs, full-obf test command output, source audit for forbidden fallback markers, and git commit hash.
- Validation command/runtime target: `env GRADLE_USER_HOME=.gradle-user-home ./gradlew :neko-test:test --rerun-tasks --tests dev.nekoobfuscator.test.JvmFullObfuscationPerfTest` plus the focused evaluator parity command.
- Completion criteria: All evaluator behavior differences are resolved or proven to be identical to the original fixture behavior; committed checkpoint exists.
