# ZKM g18 JVM State Dependency — 2026-05-15

## Scope

Harden the generated global g18 class-root helper so the class initialization root emitted for a protected class is driven by mutable JVM-side global state and class-load ordering, not only by per-class static seed material.

## Evidence

- `docs/ZKM.pdf` sections 5.2 and 6.1-6.9 describe class seed generation as a stateful g18 flow: `lookupClass()` is recorded, global node/registry state matters, and isolated recomputation may differ when class-load order changes.
- Current generated helper returns `root = projection(nodeState ^ rootMask ^ ownerHash ^ classIndex)` and only uses `globalOld` after the returned root is computed, in the mutation writes.
- Therefore the emitted class root is deterministic per class and does not strongly depend on previous protected class initialization state.
